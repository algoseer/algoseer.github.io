Future Research Ideas

This page lists a few ideas that I am either working on currently or would like to work on in the future. Most of these ideas are in a very preliminary stage, and will need more detailed thinking or planning. This page is intended to be more of a personal diary/record than a thorough documentation of my work.

RANSAC for feature selection

Random Sampling Consensus is typically used for selecting inlier samples from a dataset with a large number of outliers. It relies on the basic notion that samples that are inlier will have consensus among themselves, whereas outliers will not because they are typically random noise. RANSAC is thus often used when the there are expected to multiple inlier models or atleast there is reason to believe that the inlier characteristics have very less variance.

Can the same be extended to feature selection? Feature selection usually relies strictly on accuracy which might be unstable because of overfitting.

  • What is consensus for features? Mutual information? KL divergence of CPPs?

  • What is an inlier vs. outlier feature?

  • Feature selections vs. feature reduction

  • Consensus features or diverse features?

Mixture of Markov Chains

For a long markov process the dynamics might not stay exactly the same during the whole time. Consider, for example a meeting where different topics are being discussed. Different participants will engage in conversation differently depending on the topic. One way to model this might be to think that a particular segment is being generated by one markov chain, and the system keeps switching between these markov chains. Another way is to make the problem soft and say instead that they are being generated by a mixture of markov process.

Questions

  • Can the estimation be done in an unsupervised fashion?

  • Can this be used to automatically segement the regions according to the locally stationary properties?

Cross Lingual Tune Matching search engine

Our previous work based on tune matching employs a full signal time series analysis using DTW. While this yields reliable results, the time complexity is not well suited for real time applications.

To make this task more practical we try to tradeoff between a full time series analysis and using local statistics.

  • Use a hierarchical approach starting with the coarsest statistics

  • Go to finer statistics + some time series modeling

  • Reduce the number of hypothesis songs at each stage
